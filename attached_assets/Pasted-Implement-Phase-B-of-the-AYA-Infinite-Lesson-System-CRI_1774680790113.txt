GOAL

Allow AYA to generate large sets of exercises per lesson using AI once, then reuse them without calling AI repeatedly.

This reduces API cost while enabling effectively unlimited practice.

------------------------------------------------

Inspect:

• lesson creation pipeline

• database schema for lessons/tasks

Return short summary internally before changes.

------------------------------------------------

STEP 2 — AI LESSON CONTENT GENERATION

Create or extend a lesson generation service.

When a lesson starts:

AI should generate a batch of exercises (example: 20–40 tasks).

Each task must include:

• question
• correct answer
• optional hint
• explanation
• difficulty level
• subject tag

Start with mathematics first.

------------------------------------------------

STEP 3 — DATABASE CACHING

Store generated exercises in the database.

Example structure:

lesson_instance
exercise_set
exercise_item

When the child repeats a mission:

reuse unused tasks from the stored pool.

If pool runs out → generate a new batch.

This avoids calling AI every time.

------------------------------------------------

STEP 4 — ADAPTIVE DIFFICULTY

Connect generation to Adaptive Teacher Brain.

If child struggles:

generate easier variants.

If child performs well:

generate slightly harder tasks.
