- skill progression tracking

2. Which files are involved?

- curriculumIntegration.ts

- student progress utilities

3. Is difficulty stored anywhere such as:
- currentDifficulty
- skillLevel

- progressScore

4. Does the system already adjust:
- number size
- operation complexity
- word problem difficulty

5. Is there any logic such as:
correctStreak >= X → harder task  
wrongStreak >= X → easier task

--------------------------------
STEP 2 — DECISION
--------------------------------

If adaptive difficulty already exists:
- do NOT rewrite it
- do NOT create a second system
- only return an audit summary

If it does NOT exist:

implement a lightweight adaptive difficulty layer.

--------------------------------
STEP 3 — IMPLEMENTATION (ONLY IF MISSING)
--------------------------------

Create a small helper:

getAdaptiveDifficulty(params)

Inputs:
- grade
- topic
- recentAnswers
- correctStreak
- wrongStreak

Output:
- difficultyLevel

Difficulty levels:

1. EASY
2. NORMAL
3. CHALLENGE

Rules example:

correctStreak >= 3 → increase difficulty

wrongStreak >= 2 → decrease difficulty

--------------------------------
GRADE RULES
--------------------------------

Grade 2:

EASY
- numbers up to 20

NORMAL
- numbers up to 100

CHALLENGE
- regrouping problems
- slightly harder word problems

Grade 5:

EASY
- simple operations

NORMAL
- decimals
- fractions

CHALLENGE
- multi-step word problems
- algebra expressions

--------------------------------
INTEGRATION
--------------------------------

Before generating a task:

1. compute adaptive difficulty
2. pass difficulty to taskTemplates
3. generate task using curriculum constraints

Example flow:

difficulty = getAdaptiveDifficulty(studentState)

task = tryGenerateCurriculumTask(
  grade,
  topic,
  templateType,
  difficulty
)
