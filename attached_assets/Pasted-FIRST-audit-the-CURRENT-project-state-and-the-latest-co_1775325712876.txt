GOAL

Fix the task generation quality across AYA so that ALL generated tasks, hints, examples, and expected answers are logically consistent, age-appropriate, and aligned with the actual lesson/topic.

Current visible bug example:
Lesson shows a task like:
“Какво е червено и кръгло?”
but the hint/example talks about:
“Какво има на масата? (хляб, мляко)”
This is logically mismatched.

We need a SYSTEMIC fix, not a one-off patch.

STRICT REQUIREMENTS

Every generated task must satisfy ALL of these rules:

1. TASK-TO-HINT CONSISTENCY
The hint must help solve the exact task shown.
No generic or unrelated hint is allowed.

2. TASK-TO-EXAMPLE CONSISTENCY
The example must be from the same reasoning pattern as the task.
Example must teach HOW to solve the task, not switch to a different task type.

3. TOPIC CONSISTENCY
The task must match the topic title and subject.
Examples:
• geometry → shapes, sides, angles, forms
• division → grouping/sharing/division logic
• reading → text comprehension, sentence meaning
• memory games → memory/observation/pattern recall
Do NOT mix unrelated concepts.

4. AGE APPROPRIATENESS
Junior tasks must be simple, concrete, and child-friendly.
No abstract, confusing, or multi-layered wording for small children.

5. ANSWERABILITY
Each task must have a clear solvable answer.
No ambiguous or under-specified question.

6. NO TEMPLATE BLEED
Do NOT reuse examples/hints from another subject or another lesson type.
