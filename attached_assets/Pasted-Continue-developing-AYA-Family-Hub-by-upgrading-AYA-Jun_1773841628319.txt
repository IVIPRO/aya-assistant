Continue developing AYA Family Hub by upgrading AYA Junior into a guided AI companion learning experience for children.

Goal:
Turn AYA Junior from a simple learning module into an AI companion-based Montessori learning product for children in primary school (classes 1–4 depending on country).

Do not redesign the whole platform.
Extend the current architecture professionally and keep it scalable.

====================================
1. Core Product Direction
AYA Junior should feel like an AI learning companion for children, supervised by parents.

The experience should combine:
- AI companion character
- Montessori teaching principles
- guided learning missions
- light gamification
- parent-controlled setup

====================================
2. AYA Companion System

Create a structured companion system inside AYA Junior.

AYA is the main brand character.
Add selectable companion styles for the child:

- AYA Panda
- AYA Fox
- AYA Robot
- AYA Owl

Each companion should have:
- display name
- short personality description
- visual placeholder/avatar card
- teaching tone metadata

Example tones:
- gentle
- playful
- encouraging
- calm

The parent should be able to select the child’s companion.
The selected companion should be displayed throughout the Junior experience.

====================================

- child name
- selected AYA companion

- preferred language
- level
- XP progress
- a friendly welcome message

Example:
"Welcome back, Maggie! AYA Panda is ready for today’s learning adventure."

Add buttons:
- Enter Learning World

====================================
4. Montessori Teaching Mode

Add a Montessori-style teaching behavior layer for AYA Junior.

The AI companion should:
- encourage discovery instead of immediately giving the answer
- ask guiding questions
- break learning into small steps
- praise effort and curiosity
- use warm, simple and child-friendly language
- keep the tone calm, supportive and playful

This should be represented in the architecture and UI text so it is clear that AYA Junior follows a Montessori-inspired teaching style.

====================================

Improve the learning zones so each zone feels like a guided mission system.

For each zone, support:
- mission title
- subject
- short goal
- difficulty level
- XP reward
- completion state

Examples:
- Count to 10
- Match the right word
- Find the missing number
- Read the short story

Do not build massive content.
Create a reusable mission structure and sample mission cards for MVP.

====================================
