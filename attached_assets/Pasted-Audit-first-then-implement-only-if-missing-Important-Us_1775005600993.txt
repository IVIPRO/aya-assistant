==================================================
STEP 1 — AUDIT FIRST
==================================================

Inspect the current codebase and verify whether AYA already protects against the following:

1. Infinite render loops
   - repeated state updates
   - effects depending on changing objects/functions
   - lesson regeneration loops
   - repeated retry loops

2. Duplicate generation loops
   - same task generated repeatedly
   - same toast/celebration firing multiple times
   - repeated badge/XP awards
   - repeated progress writes on refresh/re-render

3. Navigation loops
   - child switching causing repeated reloads
   - lesson open/close loops
   - parent page refresh loops
   - dashboard auto-redirect loops

4. Speech / narration loops
   - repeated autoplay
   - overlapping playback
   - repeated “listen” triggers
   - speech state not resetting safely

5. Persistence loops / duplicate writes
   - progress saved twice
   - streak updated more than once
   - XP duplicated
   - badge history duplicated
   - achievement/timeline duplicated

6. API / fetch retry safety
   - uncontrolled retries
   - duplicate request storms
   - re-fetch on every render
   - missing guards for loading states

7. Per-child safety
   - switching child does not duplicate state
   - no state leakage
   - no shared cache mistakes

8. Fallback safety
   - app still works if analytics missing
   - app still works if lesson data partially missing
   - app still works if curriculum generator fails

Check exact files/functions involved.
