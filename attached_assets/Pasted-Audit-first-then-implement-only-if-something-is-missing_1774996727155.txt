2. Persistence scope:
   - memory only
   - component state only
   - page/session only
   - localStorage
   - backend/database

   - tied to specific child ID

3. Verify what happens after:
   - page refresh
   - route change
   - logout/login
   - switching between children

5. Check whether adaptive difficulty currently:
   - rebuilds correctly from stored answer history
   - resets unexpectedly
   - leaks between children
   - leaks between topics
   - leaks between Grade 2 and Grade 5 math

6. Identify exact files/functions currently responsible for:
   - reading saved child data
   - writing saved child data
   - restoring session state
   - restoring adaptive difficulty state

--------------------------------------------------
STEP 2 — DECISION
--------------------------------------------------

If persistence is already implemented correctly:
- do NOT rewrite anything
- do NOT add new storage
- return audit summary only

If persistence is partial or broken:

- do NOT rebuild the whole system
- reuse existing child/profile/progress structures

STEP 3 — MINIMAL FIXES ONLY IF NEEDED

- Reuse existing child profile objects
